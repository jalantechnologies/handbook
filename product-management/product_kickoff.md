# **Guide to Setting Up a New Product or Taking Ownership of an Existing Product**

---

## **Phase 1: Product Kick-Off / Transition Kick-Off**

This section outlines the steps for both **new product kick-off** and **taking ownership of an existing product**.

### 1. **MS Teams and Atlassian Tools Setup**
#### ✅ **New Product Kick-Off**:
   - Request on the **Product Team** channel to create a new **MS Team** for the product.
   - Default channels:
     - General
     - Engineering
     - Alerts
     - QA
     - Sprint Planning & Exit
   - Mention any **additional custom channels** required.
   - Request a **new Atlassian site (Jira & Confluence)** from the **Product Team**.
   - Provide an **admin email** to which admin access can be given for future user management. 

#### ✅ **Taking Ownership of an Existing Product**:
   - Ensure access to the **existing MS Team**.
   - Verify access to all **existing Jira and Confluence**.
   - If missing, request access on the **Product Team** channel.

---

### 2. **Email Alias Setup**
#### ✅ **New Product Kick-Off**:
   - Request an **email alias** via the **IT channel** (e.g., `productname@bettrsw.com`).
   - List all **team members** who need access.

#### ✅ **Taking Ownership of an Existing Product**:
   - Request access via the **IT channel** to get the **existing email alias** for the product.
   - Ensure all relevant **team members have access**.
   - If missing, request access via the **IT channel**.

---

### 3. **Miro Access for Flow Creation**
#### ✅ **New Product Kick-Off**:
   - Create a **Miro account** using the **alias email**.
   - Use the **Magic Link option** for easy access.
   - Store details in the **Tools & Credentials** file in Confluence.

#### ✅ **Taking Ownership of an Existing Product**:
   - Check if an existing **Miro account** is in place.
   - If access is missing, request it or **create a new one** with the alias email.

---

### 4. **Gather & Verify Product Documentation**
#### ✅ **New Product Kick-Off**:
   - Collect product requirements from the client.
   - Obtain:
     - Initial **scoping documents**
     - **Business Requirement Documents (BRDs)**
     - **Product Requirement Document (PRD)**
     - **Architecture Docs**
   - Pin key documents (PRD, BRD, Figma, Jira, Miro, Tools & Credentials) in the **MS Team General Channel**.

#### ✅ **Taking Ownership of an Existing Product**:
   - Locate and verify all **existing documentation**.
   - Ensure access to:
     - PRD, BRD
     - API Documentation
     - Architecture Diagrams
     - Figma Designs
   - **Update missing or outdated documents**.
   - Pin key documents in **MS Team General Channel** are not already. 

---

## **Phase 2: Development Kick-Off Preparation / Transition Readiness**

### 1. **Assemble the Team**
#### ✅ **New Product Kick-Off**:
   - Submit a request to the **Product Team Channel** specifying:
     - **Product Name**
     - **Team Requirements** (Designer, Engineer, Tech Lead)

#### ✅ **Taking Ownership of an Existing Product**:
   - Meet with the current **team composition**.
   - Put a request to the **Product Team Channel** whenever a new engineer or designer is needed on the product.

---

### 2. **Ensure Team Access to Tools**
#### ✅ **New Product Kick-Off**:
   - Request access for engineers to **Email alias** via **IT Channel**.
   - PM should invite engineers to **Jira and Confluence**.

#### ✅ **Taking Ownership of an Existing Product**:
   - Verify access to:
     - **Jira & Confluence**
     - **Source Repositories (GitHub/GitLab/Bitbucket)**
   - Request missing access via **Platform or Product Channel**.

---

### 3. **Ensure "Tools & Credentials" File Exists**
#### ✅ **New Product Kick-Off**:
   - Create a **Tools & Credentials** file in Confluence.
   - Maintain common credentials for every platform that is being used in the product. 
   - Pin the file in the **General channel** of MS Teams.

#### ✅ **Taking Ownership of an Existing Product**:
   - Locate the **existing Tools & Credentials** file.
   - Ensure it is **updated and complete**.
   - If missing, create a new one.

---

### 4. **Set Up or Verify Developer Email**
#### ✅ **New Product Kick-Off**:
   - Request the client to set up a **developer email** (`developer@product.com`).
   - Ensure it forwards to the **email alias**.
   - Use it for all platform account setups.

#### ✅ **Taking Ownership of an Existing Product**:
   - Confirm the **developer email** exists.
   - Verify that it is forwarding correctly.
   - Request email forwarding if necessary.

---

### 5. **Set Up or Validate Product Infrastructure**
#### ✅ **New Product Kick-Off**:
   - Request the **Platform Team** to set up **new infrastructure** by providing following details: 
     - Product Name
     - Internal/External Type
     - Product Email (Developer & Alias)
     - Required Repositories
     - Team Members & GitHub Handles

#### ✅ **Taking Ownership of an Existing Product**:
   - Confirm:
     - All **team members have access**.
     - CI/CD pipelines are functional.
   - Request missing infrastructure from the **Platform Team**.

---

## **Phase 3: Development Kick-Off & Operations Readiness**

### 1. **Confirm Agile Processes & Meetings**
#### ✅ **New Product Kick-Off**:
   - Schedule agile meetings **1-2 days before development starts**:
     - **Daily Stand-Up**
     - **Sprint Planning**
     - **Sprint Exit & Retro**

#### ✅ **Taking Ownership of an Existing Product**:
   - Verify **existing agile meeting schedules**.
   - Adjust meeting times if needed.

---

### 2. **Ensure PPP (Plans, Progress, Problems) Sharing**
   - Reinforce that all team members **share PPP updates** on the **General channel** every morning.
   - PM should **remind and encourage** team members to follow this process.

---

## **Final Checklist Before Full Ownership or Development Starts**
✅ MS Teams access confirmed, including all channels  
✅ Jira & Confluence access verified for all team members  
✅ Email alias and developer email access ensured  
✅ Miro account exists and is accessible via Magic Link  
✅ Product documentation is available and up-to-date  
✅ Engineers have access to repositories, CI/CD, and cloud infra  
✅ "Tools & Credentials" file is in place and updated  
✅ Agile meetings scheduled and team alignment confirmed
